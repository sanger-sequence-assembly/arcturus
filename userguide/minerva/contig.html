The Contig List

In the Projects tab, choose a project in the table and then select the File menu:

When you select the Open selected project option, Minerva displays a new table:

This table lists all of the contigs which are currently in the project that you selected.

When the table is first displayed, the contigs are sorted from largest to smallest, but you can click on the column header to sort the contigs in a different order. Hold down the SHIFT key whilst clicking on the column header to reverse the order.

What does the table show?

The table shows information about all of the contigs in the selected project, as it is stored in Arcturus at this moment.

Refreshing the display

Other programs may change the contents of a project whilst you are using Minerva. For example, a Gap4 database may be imported into Arcturus, or contigs may be moved from one project to another.

Minerva cannot automatically keep track of changes which are made by other programs.

However, you can force Minerva to update the table to show the latest situation by selecting the Refresh option in the View menu. Alternatively, simply press the F5 key at any time.

What is the contig ID?

Arcturus identifies every contig by a unique ID number. Remember that Arcturus stores information about all of the contigs that have ever existed in an assembly, so a contig's Gap4 name may not be unique. This makes the Gap4 name a very unreliable way to identify a contig.

Instead, Arcturus assigns a unique ID number to every new contig which is imported. Minerva displays this number in the column labelled ID.

Why does Minerva show the Gap4 name?

Gap4 does not yet understand Arcturus ID numbers, so Minerva also displays the Gap4 name of the contig to enable you to use Gap4 and Minerva side by side.
