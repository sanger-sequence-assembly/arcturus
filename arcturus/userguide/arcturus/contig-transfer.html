Contig transfer requests


Arcturus allows contigs to be transferred from one project to another by submitting a contig transfer request.

Minerva provides a convenient user interface for creating and managing contig transfer requests.

The contig transfer request system is best explained with an example.

The background

Alice and Bob are finishers who are working on the same assembly, but on different projects.

Alice's project is called CLONEA and Bob's project is called CLONEB.

Bob discovers that Alice's project contains a contig which he needs to make a join in his project.

The contig is identified by its unique ID number, 1234.

Bob creates a contig transfer request

Bob used Minerva to submit a contig transfer request to move contig 1234 from project CLONEA to project CLONEB.

Arcturus automatically sends an email to Alice to tell her about Bob's request.

Alice reviews the request

Alice uses Minerva to review the details of the request.

At this stage, contig 1234 is still in Alice's project, and Bob cannot acquire it without Alice's consent.

Scenario 1 : Alice refuses the request

If Alice decides that she needs contig 1234, she uses Minerva to refuse the request.

Arcturus sends Bob an email to inform him that his request has been turned down, and contig 1234 remains in Alice's project.

Scenario 2 : Alice agrees to the request

If Alice decides that she doesn't need contig 1234, she uses Minerva to approve the request.

Arcturus sends Bob an email to tell him that his request has been approved.

At this stage, contig 1234 is still in Alice's project, but either Bob or Alice can now use Minerva to execute the request.

Bob executes the request

Bob uses Minerva to execute the request.  Provided that both projects are unlocked, Arcturus moves contig 1234 from Alice's project to Bob's project, and sends an email to Alice to tell her that the contig is no longer in her project.

Contigs cannot be moved out of or into a locked project

If either project is locked when Bob tries to execute the request, Arcturus leaves the contig in Alice's project, and notifies Bob to try to execute the request again later.

Contig transfers to and from the BIN

The BIN project can never be owned by anyone.  If Bob wants to move a contig from the BIN into his project, or from his project into the BIN, he must still submit a contig transfer request, but can approve the request himself and then execute it.

This is also true for other projects which do not have an owner.

Contig transfers between two projects owned by the same person

If Bob wishes to transfer a contig from one of his projects to another of his projects, he must still submit a contig transfer request, but, as with transfers to or from the BIN or other unowned projects, Bob can approve his own request.

What happens when someone is on holiday?

Suppose that Alice has just gone away on holiday for two weeks when Bob submits his request for contig 1234.  Bob can't continue to work on his project without contig 1234.

Fortunately, Arcturus allows project coordinators and team leaders to approve contig transfer requests involving any assembly for which they are responsible.  They can even unlock someone else's project to allow a contig transfer to be executed.

Bob explains the situation to his team leader, and if she is satisfied that Alice would have approved the request, she approves it on Alice's behalf.  Arcturus notifies Alice that the request has been approved in her absence, and Bob executes the request in the usual way.

Contig transfers can sometimes fail

Occasionally, Arcturus itself will refuse to allow a contig transfer request to be approved or to be executed.

This can happen if the requested contig has ceased to be a current contig, because Arcturus does not allow old contigs to be transferred.

It can also happen if the requested contig has been moved out of its original project by another user.  This is a very unusual situation which should not occur under normal circumstances, but if finisher Chris somehow manages to move contig 1234 into his project, then when Alice tries to approve the request or Bob tries to execute it, Arcturus detects that the contig is no longer in Alice's project.  This renders the request invalid, and Arcturus marks it as failed and notifies both Alice and Bob by email.
