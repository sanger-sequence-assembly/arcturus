What is a project?


Arcturus organises an assembly by grouping sets of contigs into projects.

A project is often a set of contigs from a single chromosome or from a specific clone, but in practice, a project can contain any set of contigs that you need to group together, for whatever reason.

Ownership of a project

A project can be assigned to a specific finisher, who is said to be the owner of the project.

Ownership of an Arcturus project confers control over that project and the contigs which it contains.

The owner can lock the project to prevent other people from making changes to it.

Some software, such as the assembly pipeline, can also lock a project to prevent it from being changed whilst a critical operation is in progress.

The BIN project

Every assembly has a special project named BIN, which contains all of the contigs that have not been assigned to a chromosome or clone.

During the initial shotgun stage, the BIN is often the only project, and all new contigs will be put into it. 

Contigs are also put into the BIN by the assembly pipeline if they cannot be linked to the project for a particular chromosome or clone.

The BIN does not have an owner.

Projects and Gap4 databases

A project often corresponds to a Gap4 database.

Arcturus provides a convenient way (via the Minerva user interface) to export the contigs in an Arcturus project to a Gap4 database and to import contigs from a Gap4 database into an Arcturus project.

Whilst a project is exported as a Gap4 database, Arcturus automatically locks the project to prevent other users or programs from changing the contigs in the project.  When the Gap4 database is imported back into Arcturus, the lock is removed.

Moving contigs between projects

Contigs can be moved from one project to another by submitting a contig transfer request via Minerva.
