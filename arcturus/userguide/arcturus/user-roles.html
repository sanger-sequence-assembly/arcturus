User roles


Arcturus recognises several different types of user, which are called roles.

Arcturus grants different capabilities to each role.

Each sequencing project has its own list of users and their roles, so some people may have more than one role if they are involved in several sequencing projects.

Finisher

The most fundamental role is finisher.  A user in this role can
  • own one or more projects
  • export contigs from a project which s/he owns to a Gap4 database
  • import contigs from a Gap4 database into a project which s/he owns
  • submit a contig transfer request to move a contig to or from a project which s/he owns
  • approve another user's contig transfer request for a contig in a project which s/he owns
  • lock a project which s/he owns, to prevent anyone else from altering it

Project coordinator

A project coordinator can do all of the things which a finisher can do, and in addition, can
  • approve a contig transfer request on behalf of another user
  • change the ownership of a project
  • create a new project

Team leader

The team leader role corresponds to the Sanger Institute job title.  In Arcturus, a team leader has essentially the same capabilities as a project coordinator.

Administrator

An administrator has complete control over the entire Arcturus system and can intervene to fix problems and perform tasks which cannot be carried out using Minerva.

Annotator

An annotator has a read-only view of the contigs in an assembly.  S/he can use Minerva to view an assembly and to export contig consensus sequences in FASTA or EMBL format, but cannot alter the contigs in any way.
