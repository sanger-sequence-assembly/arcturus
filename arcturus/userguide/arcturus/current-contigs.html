Current contigs

Arcturus remembers ALL of the contigs that it has ever seen

Arcturus keeps the details of all of the contigs that have ever been loaded into it.

Unlike a Gap4 database, when you join two contigs to make a new contig, Arcturus still remembers the original two contigs.  It marks them in its internal database as the parents of the new contig.

If you now ask Arcturus for a list of contigs, it will show you the new contig but not its parents.

This is the idea behind the concept of current contigs.

The current contigs are all of the contigs which are not the parents of other contigs.

Minerva displays only the current contigs

When you use Minerva to display a list of the contigs in a project, it will show only the current contigs.

Likewise, when you use Minerva to show summary information about the projects, such as the total number of contigs, that information is based only on the current contigs.

And when you use Minerva to search for oligos or reads, it includes only the current contigs in its search.

Only current contigs can be moved between projects

The contig transfer request system will only allow current contigs to be moved between projects.

If a contig which is the subject of a transfer request ceases to be a current contig before the request is approved or executed, Arcturus will mark the request as failed and notify the requester.
