Installing Minerva on Microsoft Windows


System requirements

Minerva is a Java application.  You will need to have Java installed on your system.  Minerva requires Java 5 or higher.

You can find out whether you already have Java installed by opening the Control Panel.  If Java is already installed, there will be a control panel icon labelled "Java".

If you do not have Java on your machine, you can download Java for Windows from the Sun Microsystems Java web site.

Please seek advice from your local Windows system administrator if you are unsure how to do this, or if you do not have administrator privileges on your machine.

Downloading the software

You can download Minerva from the Sanger Institute internal web site as a Zip file.

Unpack the Zip file into a convenient directory on your machine.

Preparing to use Minerva

Minerva uses the Secure Shell (SSH) to login to the seq1 cluster to import and export Gap4 databases.

You need to follow these steps to prepare Minerva to be able to use SSH.

Step 1: Create the key files

Login to the seq1 cluster and run this command from your home directory:

ssh-keygen -t rsa -f id_rsa

When you are prompted for a passphrase, just hit the RETURN key twice.

You should see this message:

Your identification has been saved in id_rsa.
Your public key has been saved in id_rsa.pub.

Step 2: Append the public key file to your authorized_keys file

Now run this command, also from your home directory:

cat id_rsa.pub >> .ssh/authorized_keys

Step 3: Install the private key file on your Windows machine

Login to your Windows machine.

Open a command prompt window (Start --> Programs --> Accessories --> Command Prompt) and enter this command:

mkdir .ssh

The full-stop at the start of the directory name is essential.  Do not omit it, otherwise Minerva will not be able to find the private key file.

The new directory should be in the Documents and Settings\<Your Username> directory on your Windows machine.

Now copy the file named id_rsa which you created in Step 1 on the seq1 cluster into the .ssh directory which you have just created on your Windows machine.

You may find it convenient to use the Windows file manager application to copy the file from your Unix home directory and paste it into the .ssh directory on your Windows machine.

Remember that the destination .ssh directory is not in your My Documents directory on your Windows machine, but in the Documents and Settings\<Your Username> directory.

Step 4: Tidying up

You should now remove the files named id_rsa and id_rsa.pub which you created in your Unix home directory in Step 1.

Running Minerva

You can now run Minerva by double-clicking on the file named minerva.jar which you unpacked from the Zip file.

Please follow the Minerva tutorial to find out how to use Minerva.
