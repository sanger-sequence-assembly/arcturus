Contig Transfers

Minerva allows you to request the transfer of contigs from one project to another. Each request is placed in a queue for processing. If you have requested a contig which is in a project belonging to someone else, then they must give their permission before the contig can actually be transferred.

The lifecycle of a normal request

A normal contig transfer request passes through several stages. A new request is labelled as pending. If the owner of the contig gives permission for the transfer, the request's status changes to approved. At this point, the contig has not actually been transferred. It is still in its original project, but the contig's owner or the person who made the request can now execute the request. Executing an approved request moves the contig into its new project. The request's status changes to done, and the request is closed.

Refusing a request

The owner of the contig can refuse the request if s/he needs the contig to remain in its original project. The status of the request changes to refused, and the request is closed. No further action can be taken on a closed request.

Cancelling a request

The person who made the request can cancel it at any time whilst it is pending or approved. The request is removed from the queue, its status is changed to cancelled, and it is closed, so no further action can be taken on it.

How requests can fail

It is possible for a contig transfer request to fail. The most common reason is that the requested contig is no longer current, and this usually happens because the contig's original project has been imported into Arcturus after the contig was joined or split in Gap4. The request's status changes to failed and the request is closed.

A contig transfer request can sometimes be refused automatically by Arcturus. This may happen if two people try to request the same contig. The first request will be accepted, but the second request will be refused if the first request is still active (pending or approved). Arcturus allows each contig to be the subject of no more than one active request.

Contig transfers and locked projects

Arcturus will not allow a contig to be transferred into or out of a locked project. If you try to execute an approved request to or from a locked project, the transfer will not take place. The request remains in the approved state. It can be executed only when both projects have been unlocked.

Using Minerva to manage contig transfer requests

Submitting a contig transfer request

Select one or more contigs from the contig list of your chosen project, then select Transfer selected contigs to from the Contig menu.

The drop-down menu shows the projects into which you could transfer the selected contigs. In this example, you could transfer contigs to PKN6, PKNFOS17K6 or the BIN.

Viewing your contig transfer requests

Select Show all contig transfer requests from the File menu. A new display opens up:

This display lists all of your active contig transfer requests. The table in the upper panel shows the requests that you have made, plus any requests which other people have made to transfer contigs into your projects.

The table in the lower panel shows requests that other people have made for contigs in your projects.

Approving a contig transfer request

You can approve one or more pending requests by selecting the corresponding rows in the table and clicking the right mouse button over the table. A popup menu appears:

In this example, the selected requests can be cancelled or approved by selecting the appropriate menu item. If the user chooses to approve the requests, the display changes to show the new status:

Notice that the date and time has been recorded, along with the name of the person who approved the requests.

Executing a contig transfer request

You can execute one or more approved requests by selecting the corresponding rows in the table and clicking the right mouse button over the table. A popup menu appears:

Notice that it is still possible to cancel the requests at this stage. If the user chooses to execute the requests, a confirmation dialog box appears, giving the user one last chance to change his/her mind:
