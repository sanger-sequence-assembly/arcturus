The Project List

The project list shows all of the projects in the selected assembly:

What is a project?

In Arcturus, a project is a group of contigs which are being worked on by a finisher.

In most cases, it's the same thing as a "split" database, although Arcturus allows you to create new projects to enable you to group your contigs in different ways, so you could have a "split" which contains several Arcturus projects, and Minerva will display each project separately.

What does the table show?

The table shows the state of the assembly as it is stored in Arcturus at this moment.

Assembly
The name of the assembly. This will usually be the same as the name of the organism, but some organisms may have more than one assembly if the source DNA has been sequenced chromosome by chromosome.
Project
The name of the project.
Total length
The total consensus sequence length of all of the contigs in the project.
Contigs
The number of contigs in the project. This includes single reads which have been assigned to the project.
Max length
The length of the biggest contig in the project.
Reads
The total number of reads in contigs in the project. This includes single reads which have been assigned to the project.
Last contig update
The date and time of the most recent change to the project.
Owner
The name of the owner of the project. If you own any of the projects, they will be shown in bold.
Lock status
Indicates whether the project is locked, and if so, who has locked it. Contigs cannot be moved into or out of a locked project.

Sorting the rows in the table

You can sort the table by clicking on the column header which you wish to sort, using the left mouse button. For example, here is the previous display after clicking on the Max length column header:

The table is now sorted in descending order of maximum contig length, with the largest at the top.

Names are sorted alphabetically. Numbers are sorted in decreasing order, largest first. Dates are sorted in reverse date order, the most recent first.

You can reverse the sorting order by holding down the SHIFT key whilst clicking on the column header.

Altering how the information is displayed

In addition to sorting the rows of the table, you can control how the information is displayed using options in the View menu:

Refreshing the display

Other programs may change the contents of a project whilst you are using Minerva. For example, a Gap4 database may be imported into Arcturus, or contigs may be moved from one project to another.

Minerva cannot automatically keep track of changes which are made by other programs.

However, you can force Minerva to update the table to show the latest situation by selecting the Refresh option. Alternatively, simply press the F5 key at any time.

Single-read contigs

Minerva displays project summary information which includes all single reads which have been assigned to each project. Arcturus stores these reads as single-read contigs, just as Gap4 does.

You may not want to include the single-read contigs in the project summary table. To exclude them, select the Show contigs with more than one read option from the View menu. Minerva will refresh the table, showing only contigs which contain two or more reads..

To include the single-read contigs in the project summary table, select the Show all contigs option.

Project and contig dates

Arcturus stores several different kinds of date and time information about projects and contigs. Minerva alows you to choose which kind of date and time to display in the project summary table.

Project date
The table displays the last time that the ownership or lock status of the project was altered.
Contig creation date
The table displays the creation date of the newest contig in the project.
Contig updated date
Each contig has an "update" date as well as a "creation" date. When the contig is created, they are the same, but the "update" date changes whenever the contig is moved to a different project. The table displays the most recent "update" date of the contigs in the project.

Most users will not need to change this aspect of the project summary table.
